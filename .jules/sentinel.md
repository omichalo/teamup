## 2025-05-15 - Unprotected sensitive API endpoints
**Vulnerability:** The `/api/fftt/players` and `/api/teams/matches` endpoints were completely unprotected, allowing anyone to access sensitive player data (emails, phone numbers) and match details without authentication.
**Learning:** The application's `middleware.ts` explicitly excludes `/api` routes from its global protection logic (`matcher: ["/((?!api|...).*)"]`). This architectural choice requires every API route to manually implement its own security checks. These two endpoints were overlooked, possibly because they were initially thought to be public or were added during a phase where security was not the primary focus.
**Prevention:** Implement a standard security check pattern in all API routes that handle sensitive data. Added regression tests in `src/__tests__/api-security.test.ts` to ensure these endpoints remain protected. Always verify that new API routes are covered by either middleware or internal checks.
